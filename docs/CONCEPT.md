# Mineconomy — 컨셉 문서

## 개요

실제 경제 시스템을 최대한 구현하는 Minecraft Paper 플러그인.
단일 화폐(원, ₩) 기반으로 아이템 거래소, 주식 시장, 금융 시스템, 파생상품까지 포괄하는 종합 경제 플러그인.

---

## 화폐

| 항목 | 내용 |
|------|------|
| 화폐명 | 원 (₩) |
| 최소 단위 | 1원 (소수점 없음) |
| 내부 저장 | `long` 정수 |
| 수수료 소수점 | 버림 처리 |

---

## 시간 스케일

| 현실 | 게임 내 의미 |
|------|-------------|
| 1주 | 1분기 |
| 4주 | 1년 |
| 1주 | 배당 주기 (분기 배당) |
| 1주 | 이자 상환 주기 |
| 1주 | 신용등급 재산정 주기 |

## 아이템 거래소

### AMM 방식: CPMM (x·y=k)

아이템 카테고리별로 독립적인 유동성 풀 운영.

### 상장 아이템 분류

| 카테고리 | 예시 | 상장 방식 |
|----------|------|-----------|
| 블록/자원 | 돌, 흙, 모래, 나무 | 기본 상장 |
| 광물 | 철, 금, 다이아, 에메랄드 | 기본 상장 |
| 농산물 | 밀, 감자, 사탕수수 | 기본 상장 |
| 몹드롭 | 가죽, 뼈, 화약 | 기본 상장 |
| 희귀/특수 | 엘리트라, 용의숨결 | 관리자 수동 상장 |
| 무기/방어구/도구 | 검, 곡괭이, 갑옷 | 거래소 미상장 (직거래/경매장만) |
| 인챈트 아이템 | 날카로움5 검 | 거래소 미상장 (직거래/경매장만) |

### 초기 유동성

- 서버가 아이템 티어표 기반으로 초기 풀 크기 설정
- `config.yml`에서 관리자가 조정 가능
- 이후 플레이어 LP 예치 허용

### 거래 수수료

- 수수료율: 2.5%
- 수수료 행선지: 거래소 기업 수익으로 적립
- 소수점 이하 버림 처리

## 주식 시장

### AMM 방식: vAMM (가상 유동성 풀)

실제 자산 예치 없이 가상 풀로 주가 형성.

### 기업 설립

| 항목 | 내용 |
|------|------|
| 설립 주체 | 플레이어 |
| 설립 비용 | ₩500,000 |
| 최초 주식 발행량 | 1,000주 ~ 1,000,000주 (플레이어 자유 설정) |
| IPO 방식 | 공모가 설정 + 공모 기간 (최소 1 실제 시간), 누구나 참여 가능 |
| 유상증자 | 허용 (기존 주주 희석 발생) |

### 주식 종류

- 보통주: 의결권 있음
- 우선주: 의결권 없음, 배당 우선권

### 주주 권리

- 주주총회: 주요 안건 의결 (배당 여부, 합병 등)
- 의결권: 보통주 보유 비율 비례
- 배당 주기: 매 분기 (1주), 주주총회 승인 필요
- 배당 vs 자사주 매입: 경영진이 안건 상정, 주주총회 승인

### 거래소 (기본 상장 기업)

- 서버 시작 시 자동 존재하는 기본 상장 기업
- 모든 거래 수수료 → 거래소 기업 수익으로 적립
- 수익으로 자사주 매입 또는 주주 배당
- 플레이어가 거래소 주식 매매 가능

### 상장폐지 조건

- 기준: 초기 시가총액의 5% 미만
- 유예기간: 7일 (1주)
- 유예기간 내 회복 못하면 자동 상장폐지
- 주주총회로 막을 수 없음

### 가격 제한

| 항목 | 기준 |
|------|------|
| 상하한가 | 개별 종목 ±30% (즉시 당일 거래 종료) |
| 서킷브레이커 | 전체 인덱스 ±20% → 당일 전체 거래 종료 |

## M&A

### 일반 M&A

- 시장에서 주식 매집으로만 처리
- 50% 초과 보유 시 경영권 자동 획득
- 피인수 기업은 자회사로 전환 (상장 유지)
- 완전 합병: 양사 주주총회 동시 승인 필요

### 적대적 M&A

- 허용
- 상대 기업 주식을 시장에서 몰래 매집 가능
- 50% 초과 시 경영권 자동 탈취
- 방어 수단: 포이즌 필 (기존 주주에게 할인 발행 → 매수자 지분율 희석)

## 금융 / 은행

### 대출

- 일반 대출
- 담보 대출 (아이템 또는 주식)
- 금리: 기준금리(관리자 설정) + 신용등급별 가산금리
- 이자 상환 주기: 매 분기 (1주)

### 신용등급

| 등급 | 대출 한도 | 가산금리 |
|------|-----------|----------|
| A+ | 보유 자산의 500% | +0% |
| A | 보유 자산의 300% | +1% |
| B+ | 보유 자산의 200% | +3% |
| B | 보유 자산의 150% | +5% |
| C+ | 보유 자산의 100% | +8% |
| C | 보유 자산의 50% | +13% |
| D | 대출 불가 | - |

- 신규 플레이어 기본 등급: B
- 신용등급 재산정 주기: 매 분기 (1주)

## 공매도 / 신용거래

| 항목 | 기준 |
|------|------|
| 신용매수 최소 증거금 | 보유 자산의 40% |
| 공매도 증거금 | 매도 금액의 105% |
| 신용매수 강제 청산 | 증거금 비율 20% 미만 |
| 공매도 강제 청산 | 매도가 대비 50% 이상 상승 |
| 숏스퀴즈 | 허용 |
| 개인 공매도 한도 | 없음 |

## Vault 호환

- `Economy` 인터페이스를 얇게 구현해 화폐 잔액만 노출
- AMM, 주식, 파생상품 기능은 별도 Mineconomy API로 제공
- `mineconomy-api` 모듈을 통해 외부 플러그인 연동
