# Mineconomy

[English](./README.md) | [한국어](./README.ko.md)

Mineconomy는 Minecraft 서버에 실제 경제 시스템을 최대한 구현하는 오픈소스 플러그인입니다. 단일 화폐(원, ₩) 기반으로 AMM 아이템 거래소, 주식 시장, 금융, 파생상품까지 포괄하는 종합 경제 플러그인입니다.

## 기능 목록

### &#9744; **화폐 시스템**:
- 단일 화폐 **원 (₩)**, `Long` 정수 저장, 소수점 없음.
- 거래 수수료 2.5% (버림), 거래소 기업 수익으로 적립.
- 사망 패널티: 잔액 3% 소실; 장기 오프라인(4주+): 자동 청산.

### &#9744; **아이템 거래소 (AMM · CPMM)**:
- 카테고리별 독립 유동성 풀, **x·y=k** 공식 적용.
- 기본 상장: 블록, 광물, 농산물, 몹드롭.
- 관리자 수동 상장: 희귀 아이템 (엘리트라, 용의 숨결 등).
- 무기·방어구·인챈트 아이템은 직거래 / 경매장만 지원.
- 플레이어 LP 예치 가능; 초기 풀 크기는 `config.yml`에서 설정.

### &#9744; **주식 시장 (vAMM)**:
- 실제 자산 예치 없는 가상 유동성 풀로 주가 형성.
- 플레이어가 기업 설립: 설립비 ₩500,000, 최초 발행 1,000~1,000,000주.
- IPO: 공모가 설정 + 공모 기간 (최소 1 실제 시간).
- 보통주(의결권) / 우선주(배당 우선권).
- 주주총회: 배당, 자사주 매입, 합병 등 주요 안건 의결.
- 개별 종목 ±30% 상하한가, 전체 인덱스 ±20% 서킷브레이커.
- 상장폐지: 초기 시가총액의 5% 미만 1주 지속 시 자동 폐지.

### &#9744; **M&A**:
- 시장에서 50% 초과 매집 시 경영권 자동 획득.
- 적대적 M&A 허용; 포이즌 필 방어 수단 지원.
- 완전 합병: 양사 주주총회 동시 승인 필요.

### &#9744; **공매도 / 신용거래**:
- 신용매수: 보유 자산의 40% 최소 증거금.
- 공매도: 매도 금액의 105% 증거금.
- 강제 청산: 증거금 비율 20% 미만(신용매수) / 매도가 대비 50% 이상 상승(공매도).
- 숏스퀴즈 허용, 개인 공매도 한도 없음.

### &#9744; **금융 / 은행 시스템**:
- 일반 대출 및 담보 대출 (아이템·주식).
- 금리 = 기준금리(관리자 설정) + 신용등급별 가산금리.
- 이자 상환 주기: 분기 (매 1주).
- 신용등급 A+~D, 분기마다 재산정.

### &#9744; **파생상품** *(v2 예정)*:
- **선물**: 1/2/4분기 만기, 차액 정산, 롤오버 가능.
- **옵션**: 미국형, AMM 방식 프리미엄, 기초자산: 아이템·주식.

### &#9744; **시장 지표**:
- 시가총액 가중평균 전체 인덱스, 시가총액 순위.
- 차트 단위: 1분 / 15분 / 1시간 / 1일 / 1주 / 전체.

### &#9744; **관리자 도구**:
- 기준금리, 초기 풀 유동성, 아이템 티어표 설정.
- 서버 전체 경제 통계 실시간 대시보드.

### &#9744; **GUI 지원**:
- 커스텀 모델 데이터 방식 체스트 UI 직접 구현.
    - [ ] **거래소 GUI**: AMM 풀 거래 인터페이스.
    - [ ] **주식 GUI**: 호가창 및 차트 뷰어.
    - [ ] **은행 GUI**: 대출 내역 및 신용등급 현황.

### &#9745; **Vault 호환**:
- `Economy` 인터페이스를 얇게 구현해 화폐 잔액만 노출.
- Mineconomy 고유 기능은 `mineconomy-api` 모듈로 제공.

## 설치 방법
1. [플러그인 다운로드](https://github.com/Nekonic/Mineconomy/releases/) 후 `plugins` 폴더에 추가합니다.
2. [Vault](https://www.spigotmc.org/resources/vault.34315/)를 의존 플러그인으로 설치합니다.
3. 리소스팩 URL을 복사하여 `server.properties`의 `resource-pack=` 뒤에 붙입니다.
4. `require-resource-pack=true`로 변경합니다.
5. 리소스팩 파일을 SHA-1로 해시하여 `resource-pack-sha1=`에 입력합니다.

Windows:
```shell
certutil -hashfile .\Mineconomy-Resource-Pack.zip sha1
```
Linux:
```bash
sha1sum Mineconomy-Resource-Pack.zip
```
6. `config.yml` 파일을 통해 초기 설정을 완료합니다.
7. 서버를 재시작하거나 `/reload confirm` 명령어를 사용해 플러그인을 활성화합니다.
